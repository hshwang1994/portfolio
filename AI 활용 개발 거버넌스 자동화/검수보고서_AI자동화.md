## 4. 서사-코드 불일치 정정

- **D1. 머지 후 검사 c (외부 enum)**: 포트폴리오 §4.2 사례2, §5.5, §9.5는 "검사 c가 외부 enum 영향 범위를 자동 탐색"한다고 서술했으나, 실제 `check_external_contract_enums()`는 **rule 96 R1 폐기로 비활성(빈 결과 반환)**. → "origin 자동검사는 폐기 후 비활성, 외부 계약은 사람 질의 게이트(rule 96 R2/R3/R4)로 이관"으로 정정. 8종 검사 헤더도 "7 활성 + c 비활성"으로 명시.
- **D2. Tier A 자동 수정(Spotless)**: "import 순서 정렬 (Spotless 100% 자동)"이 활성처럼 표기됐으나 yaml에서 **주석 처리(미도입)** → "Spotless, 도입 후보"로 정정.
- **D3. Q13 면접 답변**: `plugin-B` 신규 모듈 + `ExtendedPhaseType`을 만들었다는 서술이 **존재하지 않는 모듈**(고객사 B는 plugin 없이 yaml.profile로 분기, §9.4, settings.gradle 확인) → "core enum 미변경 + profile 설정 분리"로 아키텍처 일치하게 재작성.
- **D4. Q27 자기기여**: "자동화 영역 100% 단독" + "나머지 17.4%는 본인 백엔드"는 오귀속(자동화영역 19커밋은 타 개발자) → "설계, 구현 단독 주도 + 커밋 82.7%, 나머지 19건은 타 개발자 머지, 수정"으로 정정.
